HTML Forms

Lecture 6 • Reading time: 10 minutes

HTML forms are used to collect user input and send data to a server. This lecture covers the form element, inputs, labels, validation attributes, and how form submission works.

Introduction to HTML Forms

Forms are used to collect information from users. The data is usually sent to a server for processing (e.g. saving to a database or sending an email).

Common examples include:

  • Login forms
  • Registration forms
  • Feedback forms
  • Search forms

Form data is typically submitted to a server-side script or API that handles validation, storage, and response.

The Form Element

The <form> element acts as a container for form controls (inputs, buttons, etc.). It defines where the data goes and how it is sent. 

<form action="/submit" method="post">
<!-- form elements -->
</form>

Key attributes:

  • action — the URL where form data is sent
  • method — the HTTP method used (GET or POST)

Form Attributes

Attribute Description
actionURL where form data is sent
methodGET or POST — how data is sent
targetWhere the response is displayed (e.g. same window, new tab, iframe)

GET vs POST: GET appends data to the URL (visible in the address bar; good for searches). POST sends data in the request body (not in the URL; better for passwords and larger or sensitive data).

Basic Form Structure

Code explanation: This example shows a complete page with a simple form: a label and a text input. The for attribute on <label> matches the input’s id, so clicking the label focuses the field. Run the code to see the form in the browser.

Example

The Label Element

The <label> element associates text with a form input. Use the for attribute set to the input’s id so that clicking the label focuses the input. 

<label for="email">Email:</label>
<input type="email" id="email" name="email">

When the user clicks the label text (“Email:”), the browser focuses the associated input field. This is especially helpful on small screens and for accessibility.

The Input Element

The <input> element is the most common form control. It supports many input types (text, password, email, number, etc.) and can be styled and validated with attributes. 

<input type="text" name="username">

The type determines how the input looks and behaves; name identifies the field when the form is submitted.

Common Input Types

Input Type Description
textSingle-line text input
passwordHidden password input (characters masked)
emailValidates email format
numberNumeric input
dateDate picker

<html>
<head><meta charset="utf-8"><title>Form Controls Example</title></head>

Checkbox Input

Checkboxes are used when users can select multiple options. Each checkbox has its own name (or the same name for a group); the value is sent when the box is checked. 

<input type="checkbox" id="subscribe" name="subscribe">
<label for="subscribe">Subscribe to newsletter</label>

Radio Buttons

Radio buttons are used when the user must select only one option from a group. All options in the group share the same name; each has a different value. 

<input type="radio" name="gender" value="male"> Male
<input type="radio" name="gender" value="female"> Female

Select Dropdown

The <select> element creates a dropdown menu. Each <option> is one choice; the value is sent when that option is selected. 

<select name="country">

<option value="us">United States</option>
<option value="uk">United Kingdom</option>

</select>

Textarea Element

The <textarea> element is used for multi-line text input (e.g. comments or messages). You can set the size with rows and cols, or with CSS. 

<textarea name="message" rows="4" cols="50">
Enter your message
</textarea>

Button Element

The <button> element creates a clickable button. The type attribute controls its behavior:

  • submit — submits the form
  • reset — resets form fields to their initial values
  • button — no default action (use with JavaScript)
<button type="submit">Submit</button>

Form Validation Attributes

HTML5 provides attributes that let the browser validate input before submission:

Attribute Purpose
requiredField must be filled before submit
maxlengthLimits maximum input length
patternValidates input against a regular expression

Example:

<input type="text" name="username" required maxlength="10">

Placeholder Attribute

The placeholder attribute shows hint text inside the input when it is empty. It disappears when the user types. It does not replace a proper <label> for accessibility. 

<input type="text" placeholder="Enter your username">

Name and Value Attributes

The name attribute identifies the input when the form is sent to the server. Each field that should be submitted needs a name. The server receives name–value pairs (e.g. username=JohnDoe).

The value attribute sets the default value of an input. For text fields it’s what appears in the box; for buttons it’s the submitted value when that button is used. 

<input type="text" name="username" value="JohnDoe">

Form Submission Process

How form submission works:

  1. User fills form fields — enters data in inputs, selects options, etc.
  2. Browser sends data to server — when the user submits (e.g. clicks Submit), the browser collects name–value pairs and sends them using the form’s action and method.
  3. Server processes the request — the server (or server-side script) validates the data, saves it, sends email, etc.
  4. Response is returned to the browser — the server sends back a new page, a redirect, or JSON so the user sees a result or confirmation.

Form Enhancements with JavaScript

JavaScript can improve forms without replacing server-side validation. Examples:

  • Real-time validation — check fields as the user types and show errors before submit
  • Dynamic form fields — add or remove fields, show/hide sections based on previous answers
  • Interactive behavior — auto-format input, suggest values, or prevent submit until required fields are valid

Always validate on the server as well; client-side validation improves UX but can be bypassed.

Lecture Summary

Key Takeaways

  • HTML forms collect user input and send it to a server.
  • The <form> element contains form controls and defines action and method.
  • Input elements provide various data types (text, email, number, checkbox, radio, etc.).
  • Validation attributes (required, maxlength, pattern) improve data accuracy.
  • Forms communicate with servers to process data; JavaScript can enhance the experience.
