CSS Introduction

Lecture 7 • Reading time: 10 minutes

CSS is the language used to style HTML pages and control layout. This lecture introduces CSS syntax, common properties, ways to apply styles, the box model, and specificity.

What is CSS?

CSS stands for Cascading Style Sheets. It is used to control the presentation of web pages.

CSS is used to control:

  • Colors — text, backgrounds, borders
  • Fonts — typeface, size, weight
  • Layout — position of elements, columns, grids
  • Spacing — margin, padding
  • Positioning of elements — normal flow, flexbox, grid, or absolute/fixed

CSS allows you to style multiple web pages at once by linking one or more external stylesheets, so you can keep a consistent look and make global changes in one place.

Why CSS is Important

Benefits of using CSS:

Benefits include:
  • Separation of design and structure — HTML for content, CSS for appearance
  • Reusable styles — define once, apply to many elements or pages
  • Faster website maintenance — change one file to update the look site-wide
  • Consistent design across pages — shared stylesheets keep branding and layout unified

CSS Syntax

A CSS rule consists of a selector and a declaration block. The selector chooses which element(s) to style; the block contains one or more property: value; declarations.

h1 {
  color: red;
  text-align: right;
}

Here h1 is the selector. Inside the braces: color is the property, red is the value. Each declaration ends with a semicolon.

Common CSS Properties

Property Description
colorSets text color
font-familyDefines font type
font-sizeControls text size
background-colorSets background color
marginControls outer spacing (outside the border)
paddingControls inner spacing (inside the border)

Ways to Use CSS

There are three ways to apply CSS to HTML:

Type Where Written Best Use
Inline CSSInside HTML tag (style attribute)Small, one-off changes
Internal CSSInside <style> tag in the pageSingle-page styling
External CSSSeparate .css fileLarge projects, multiple pages

Inline CSS Example

Inline style in the style attribute. Run it to see red text.

Example
<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<title>Inline CSS</title>
</head>
<body>
<p style="color:red;">This is red text</p>
</body>
</html>

Internal CSS Example

Internal CSS in a <style> tag. Run it to see the result.

Example
<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<title>Internal CSS</title>
<style>p { color: blue; }</style>
</head>
<body>
<p>This paragraph is styled with internal CSS.</p>
</body></html>

External CSS Example

External CSS is stored in a separate file and linked with <link>. This is the preferred approach for multi-page sites.

In your HTML (e.g. in <head>):

<link rel="stylesheet" href="style.css">

In style.css:

p {
  color: green;
}

Typeface vs Font

Typeface is the design of the characters (e.g. Arial, Times New Roman). Font is a specific size and style of a typeface (e.g. Arial Bold 14px).

In CSS you often set several font properties at once with the shorthand font:

font: bold 40px sans-serif;

Types of Fonts

Font Type Description
SerifDecorative edges (serifs) on letters; traditional, print-like
Sans-serifClean, modern fonts without serifs
MonospaceEqual character width; often used for code
CursiveHandwriting-style fonts
FantasyDecorative, display fonts

Serif vs Sans-serif

Serif fonts have small strokes (serifs) at the ends of letters; they are common in print and can look formal. Sans-serif fonts do not have these strokes; they look clean and minimal.

Sans-serif fonts are usually easier to read on screens, especially at smaller sizes. Serif fonts are often used for headings or long print-like text. For body text on the web, sans-serif (e.g. Arial, Helvetica, system fonts) is very common.

CSS Box Model

Every HTML element is treated as a box. The box model describes how content, padding, border, and margin combine to form the total size and spacing of an element.

Components (from inside to outside):

  • Content — the text or other content inside the element
  • Padding — space between the content and the border
  • Border — the edge around the padding
  • Margin — space outside the border (between this element and others)

Box Model Example

Box model: content, padding, border, margin. Run it to see the layout.

Example
<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<title>Box Model</title>
<style>div { border: 2px solid #333; padding: 20px; margin: 10px; }</style>
</head>
<body>
<div>Content with border, padding, and margin</div>
</body></html>

Styling Tables with CSS

You can style tables with borders, padding, and collapse so cells share borders:

table {
  border: 1px solid black;
  border-collapse: collapse;
}

td, th {
  border: 1px solid black;
  padding: 5px;
}

Identifying Elements in CSS

CSS selectors target elements so you can apply styles. Common ways to identify elements:

  • Element (tag) selector — div, span, p, h1 — targets all elements of that type
  • ID selector — #foo — targets the single element with id="foo"
  • Class selector — .baz — targets all elements with class="baz"

ID vs Class

Use id for a unique element on the page; use class when multiple elements share the same style or behavior.

Example HTML:

<h1 id="foo">Heading1</h1>
<h1 class="baz">Heading3</h1>

Example CSS:

#foo {
  color: blue;
}

.baz {
  color: red;
}

CSS Specificity

When more than one rule could apply to the same element, the browser uses specificity to decide which style wins. From highest priority to lowest:

  1. Inline styles (in the style attribute)
  2. ID selectors (#id)
  3. Class selectors (and attributes, pseudo-classes)
  4. Element selectors (and pseudo-elements)

More specific selectors override less specific ones. In case of a tie, the rule that appears later in the CSS wins.

Specificity Example

h1 {
  color: red;
}

#foo {
  color: blue;
}

For an element <h1 id="foo">, both rules apply. The ID selector #foo has higher specificity than the element selector h1, so the heading will be blue. The ID overrides the element selector.

Lecture Summary

Key Takeaways

  • CSS styles HTML pages: colors, fonts, layout, spacing, and positioning.
  • CSS rules consist of selectors and declaration blocks (property: value).
  • CSS can be applied inline, internally (in-page <style>), or externally (linked .css file).
  • The box model (content, padding, border, margin) controls element spacing and size.
  • CSS specificity determines which style is applied when rules conflict; inline > ID > class > element.
