Tables, Div, Span and Layout Design in HTML

Lecture 4 • Reading time: 9 minutes

HTML provides elements to structure webpage layout and organize content effectively. This lecture covers tables for data, and the <div> and <span> elements used in modern layout design.

Introduction to HTML Layout

HTML provides elements used to structure webpage content. To design layout and organize content, three elements are especially important:

  • HTML Tables — for tabular data in rows and columns
  • Div — a block-level container for sections
  • Span — an inline container for styling small parts of text

Together, these elements help organize webpage content and create a clear layout structure.

What Are HTML Tables?

HTML tables are used to display data in rows and columns. They are mainly used for tabular data, such as:

  • Price lists
  • Student records
  • Schedules
  • Statistics

In modern web design, tables should not be used for general webpage layout. Use CSS and <div> (or semantic HTML5 elements) for layout instead.

Basic Table Structure

Simple table with header and data rows. Run it to see how it looks.

Example
<table>
<tr>
<th>Header 1</th>
<th>Header 2</th>
</tr>

<tr>
<td>Data 1</td>
<td>Data 2</td>
</tr>
</table>

HTML Table Elements

These elements work together to build a table:

Tag Description
<table>Defines a table
<tr>Defines a row
<th>Table header cell (bold by default)
<td>Table data cell

<table> wraps the whole table. Each <tr> is a row; inside it, <th> is for headers and <td> for data cells.

Table Attributes

Traditional table attributes used for formatting (styling is now often done with CSS):

Attribute Purpose
borderAdds borders to the table
cellpaddingSpace inside cells
cellspacingSpace between cells

Example:

<table border="1" cellpadding="5" cellspacing="5">

Note: For new projects, prefer CSS (border, padding, border-collapse) instead of these attributes.

Colspan and Rowspan

colspan and rowspan merge cells. Run it to see the table.

Example
<!DOCTYPE html>
<html>
<head><meta charset="utf-8"><title>Colspan and Rowspan</title></head>
<body>
<table border="1">
<tr><td colspan="2">Merged Column</td></tr>
<tr><td rowspan="2">Merged Row</td><td>A</td></tr>
<tr><td>B</td></tr>
</table>
</body></html>

Using Tables for Layout (Deprecated)

Older websites often used tables to structure the whole page (header, main content, footer):

<table>
<tr>
<td>Header</td>
</tr>

<tr>
<td>Main Content</td>
</tr>

<tr>
<td>Footer</td>
</tr>
</table>
Warning Using tables for layout is outdated and not recommended. Use <div> (or semantic elements like <header>, <main>, <footer>) with CSS for layout instead.

Introduction to Div

The <div> element is a block-level container used to group and structure sections of a webpage. It has no default meaning; you give it meaning with id, class, and CSS.

Common uses:

  • Header sections
  • Content areas
  • Footers
  • Sidebars

Div Example

Div groups the heading and paragraph. Run it to see the result.

Example
<!DOCTYPE html>
<html>
<head><meta charset="utf-8"><title>Div Example</title></head>
<body>
<div>
<h1>Website Title</h1>
<p>This is a paragraph inside a div.</p>
</div>
</body></html>

Div Layout Structure

Three sections: header, content, footer. Run it to see the layout.

Example
<!DOCTYPE html>
<html>
<head><meta charset="utf-8"><title>Div Layout</title></head>
<body>
<div id="header">Header</div>
<div id="content">Main Content</div>
<div id="footer">Footer</div>
</body></html>

Introduction to Span

Span is for styling small bits of text. Run it to see it.

Example
<!DOCTYPE html>
<html>
<head><meta charset="utf-8"><title>Span Example</title></head>
<body>
<p>This is a <span>highlighted</span> word.</p>
</body></html>

Div vs Span

Feature Div Span
TypeBlock elementInline element
Line breakYes (starts on new line)No
UsagePage sections, layoutStyling text within a line

Layout Using Div (Modern Method)

Modern websites use nested <div> elements (and semantic elements) with CSS instead of tables:

<div class="container">

<div class="left">
Left Section
</div>

<div class="right">
Right Section
</div>

</div>

The outer container div wraps the layout. The inner left and right divs can be styled with CSS (e.g. flexbox or grid) to sit side by side or stack on small screens.

Layout Best Practices

Best Practices
  • Use tables only for tabular data
  • Use div (or semantic elements) for webpage layout
  • Use span for styling small parts of text
  • Avoid tables for layout design

Lecture Summary

Key Takeaways

  • Tables display tabular data in rows and columns.
  • Div structures webpage sections and is the standard for layout.
  • Span styles inline text without breaking the line.
  • Modern layouts use div (and semantic HTML) instead of tables.
  • Proper structure improves maintainability and accessibility.
