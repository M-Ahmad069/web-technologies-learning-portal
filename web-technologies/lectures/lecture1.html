Introduction to Web Platforms

Lecture 1 • Reading time: 8 minutes

Web platforms form the foundation of how we access and interact with applications over the internet. This lecture introduces the client-server model, the role of web browsers, and how requests and responses flow between users and servers.

Topics Covered

Topics Covered:

  • What is a web platform
  • Client-server architecture
  • Web browsers and their role
  • HTTP request–response cycle
  • Front-end vs back-end

What is a Web Platform

A web platform is the underlying software and infrastructure that enables applications and content to be delivered and used over the internet. Understanding this concept is essential for anyone building or using web technologies.

Definition

A web platform is a software system that allows users to access and interact with applications and content over the internet through a browser or other client, typically using standardized protocols such as HTTP and HTTPS.

Key Components of Web Platforms

Web platforms are built from several core components that work together to deliver content and functionality.

Infrastructure

Servers, networks, and hosting that store and deliver web content and applications.

Development Tools

Frameworks, editors, and APIs used to build and maintain web applications.

Content Management

Systems for creating, storing, and publishing digital content.

User Interface

The visual and interactive layer that users see and interact with in the browser.

APIs

Application Programming Interfaces that allow different systems to communicate and share data.

Security Features

Authentication, encryption, and policies that protect data and users.

Web Communication (HTTP)

HTTP (HyperText Transfer Protocol) is the foundation of data communication on the web. When you open a webpage, your browser sends an HTTP request to a server, and the server responds with the requested content.

Browser → Request → Server
Server → Response → Browser

HTTP Request-Response Cycle

The cycle of communication between client and server follows these steps:

  • The user enters a URL or clicks a link in the browser.
  • The browser constructs an HTTP request and sends it to the server.
  • The server receives the request, processes it, and locates the requested resource.
  • The server sends back an HTTP response containing the resource (e.g. HTML, CSS, or data) and a status code.
  • The browser receives the response and renders the content for the user.

HTTP Methods

HTTP defines several methods that indicate the desired action to be performed on a resource.

Method Description
GETRetrieve data
POSTSubmit data
PUT / PATCHUpdate existing data
DELETERemove data

HTTP Status Codes

Every HTTP response includes a status code that indicates whether the request succeeded or how it failed.

HTTPS Security

HTTPS (HTTP Secure) is the encrypted version of HTTP. It uses SSL/TLS (Secure Sockets Layer / Transport Layer Security) to encrypt data between the browser and server, protecting it from eavesdropping and tampering.

Encryption: HTTPS encrypts the data in transit so that even if someone intercepts the connection, they cannot read or modify the content. SSL/TLS certificates also verify that you are connected to the real server.

Security benefits of HTTPS:

  • Encryption of data in transit
  • Authentication of the server (via certificates)
  • Integrity: detection of modified data
  • Trust and better ranking in search engines

Web Standards

Standards ensure that the web works consistently across browsers and devices. Key organizations include:

W3C

World Wide Web Consortium—develops standards for HTML, CSS, and many web technologies.

IETF

Internet Engineering Task Force—develops and promotes standards such as HTTP and TCP/IP.

WHATWG

Web Hypertext Application Technology Working Group—maintains the HTML Living Standard.

Web Constraints

Building for the web involves working within various constraints that affect design and implementation.

Technical Constraints

Browser compatibility, performance, network latency, and device capabilities.

Legal Constraints

Privacy laws (e.g. GDPR), accessibility requirements, and intellectual property.

Design Constraints

Screen sizes, input methods, and user expectations for usability and accessibility.
